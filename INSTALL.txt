============================================================
  AUDIO TO SRT
  Transcribe audio & add subtitles in DaVinci Resolve
  Powered by ElevenLabs Scribe AI
============================================================


WHAT YOU NEED BEFORE STARTING
------------------------------
  1. DaVinci Resolve  (Free or Studio)
     https://www.blackmagicdesign.com/products/davinciresolve

  2. Python 3.10 or newer
     https://www.python.org/downloads/
     (Windows: setup.bat can install it for you automatically.
      If you install manually, check "Add Python to PATH".)

  3. ElevenLabs account  (free tier works)
     Sign up and get your API key at:
     https://elevenlabs.io/app/speech-synthesis/api


QUICK SETUP  (recommended)
---------------------------

  MAC
  ---
  1. If you downloaded a ZIP, double-click it to extract first,
     then put the extracted folder anywhere you like
     (e.g. Documents, Desktop — doesn't matter)

  2. Double-click  setup.command
     If Mac says "cannot be opened", right-click it → Open → Open.
     On newer macOS there is no Open button: go to
     System Settings → Privacy & Security, scroll down and click
     "Open Anyway", then confirm.
     (Last resort: open Terminal, type  bash  and a space, drag
      setup.command into the window, press Enter.)

  3. Follow the prompts — enter your ElevenLabs API key when asked

  4. Done!


  WINDOWS
  -------
  1. If you downloaded a ZIP: right-click it → "Extract All..." FIRST.
     Running setup from inside the un-extracted ZIP will not work.
     Put the extracted folder anywhere you like
     (e.g. C:\Users\YourName\Documents)

  2. Double-click  setup.bat
     If SmartScreen shows "Windows protected your PC", click
     "More info" → "Run anyway".

  3. Follow the prompts — enter your ElevenLabs API key when asked
     (If Python is missing, setup offers to install it automatically.)

  4. Done!


HOW TO USE
----------
  1. Open DaVinci Resolve
  2. Open a project and go to the Edit page
  3. Make sure your timeline has an audio clip on it
  4. Click the menu:  Workspace  →  Scripts  →  audio_to_srt
  5. The Audio2SRT window appears. Pick the audio track, language and
     caption style, tweak anything else you like, then click
     "Generate SRT".
  6. Wait while ElevenLabs transcribes the audio (internet required)
  7. (optional) Review/edit the captions when the review list appears
  8. Captions are automatically added to the timeline


THE MAIN WINDOW — WHAT'S THERE
------------------------------
  Subtitle settings   max chars / lines / seconds, minimum on-screen
                      time, reading-speed cap (CPS) and text size
  Language            11 Indian languages. Hindi guarantees Devanagari
                      output — even spoken English words
  Caption style       "SRT (colored)" = classic subtitle track with
                      per-speaker colours. "Plain Text" = unstyled.
                      Everything else = animated Fusion Text+ captions
                      (Karaoke, Fade, Pop In, Slide Up, ...)
  Options             Punctuation, Speaker detection (diarization),
                      Censor words (uses censor_words.txt)
  Presets             save/load all your settings under a name
  Font                any installed font + weight
  Reel styles         one-click looks for vertical reels
  Advanced            max words per caption, keyword highlight colour,
                      review-before-applying
  Silence cut         renders a "silence removed" copy of the clip +
                      matching SRT into the Media Pool (needs ffmpeg,
                      see below). The timeline itself is never touched.
  Preview panel       live animated preview + position/size/colour
                      controls for the animated styles

  Update Captions     restyles the animated captions already on the
                      timeline (no re-transcription, free)
  Undo Last Captions  removes the caption track the last run created


AUTO-CUT SILENCE  (optional)
----------------------------
  Turn on "Cut silence (tightened copy)" in the Silence Cut section.

      Threshold dB  how quiet counts as silence (-30 default)
      Min gap       ignore gaps shorter than this, seconds (0.5 default)
      Pad           breath left around speech, seconds (0.05 default)

  The script renders a copy of the clip with the silent gaps removed,
  plus a matching SRT, and imports both into your Media Pool. Your
  timeline is NOT changed — drop the tightened clip onto a fresh track.

  Requires ffmpeg (Mac: brew install ffmpeg). Subtitles work without it.


SUBTITLE SETTINGS EXPLAINED
----------------------------
  The three classic numbers (Max chars / Max lines / Max sec) mean:
    25  →  max characters per line
     1  →  max lines on screen at once
     2  →  max seconds per subtitle cue

  Examples:
    25,1,2   →  short single-line captions (default — Hindi reel style)
    42,2,7   →  two-line captions, longer phrases (English style)


TROUBLESHOOTING
---------------
  Script doesn't appear in Resolve menu
  → Quit and reopen Resolve after setup
  → On Windows, setup installs to your user script folder first:
      %APPDATA%\Blackmagic Design\DaVinci Resolve\Support\Fusion\Scripts\Utility

  "Python was not found; run without arguments to install from the
   Microsoft Store..."
  → You are running an OLD copy of setup.bat. Download the latest
    version and run its setup.bat — it finds or installs Python
    correctly and ignores the fake Microsoft Store "python".

  "No module named elevenlabs"
  → Run setup again, or manually:
      Mac:     pip3 install -r requirements.txt
      Windows: pip install -r requirements.txt

  No dialog appears when running the script
  → tkinter is missing from your Python.
      Mac:     brew install python-tk
      Windows: re-run the Python installer and keep "tcl/tk and IDLE"
               checked.
    Details are logged in  logs/audio_to_srt.log.dialog

  "ELEVENLABS_API_KEY not set"
  → Open the .env file in this folder and add:
      ELEVENLABS_API_KEY=your_key_here

  "No active timeline" error
  → Make sure you're on the Edit page and a timeline is open

  Subtitles are out of sync
  → This script syncs to the first clip on the selected audio track
  → If that clip is trimmed, the script now uses the clip's source in/out
  → If the track contains multiple separate clips, run the script on the clip you want as the first clip on that track

  Still stuck?
  → Check the log files in the  logs/  folder inside this directory:
      audio_to_srt.log            main script log
      audio_to_srt.log.transcribe transcription output & errors
      audio_to_srt.log.dialog     dialog / tkinter errors
